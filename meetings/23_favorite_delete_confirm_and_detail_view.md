# 23. Favorite Delete Confirm and Detail View

## 中文

- 删除收藏不再使用浏览器原生确认框。
- 点击删除后，会在删除按钮下方出现小确认窗，点击 `Confirm` 才会真正删除。
- 收藏列表里的操作改为明确的 `查看` 和 `删除`。
- 点击查看会打开收藏详情，默认是阅读模式，不再直接进入编辑状态。
- 收藏详情标题行显示收藏名称，右侧同行提供 `个性化`、`删除` 和关闭按钮。
- 详情里的删除按钮也使用同样的小确认窗逻辑。
- 点击 `个性化` 后才进入编辑模式，可编辑名称、简介、原料和做法。

## English

- Favorite deletion no longer uses the native browser confirmation dialog.
- Clicking delete now opens a small confirmation panel below the delete button; the favorite is removed only after clicking `Confirm`.
- Favorite list actions are now explicit: `View` and `Delete`.
- Clicking view opens the favorite detail modal in read-only mode by default.
- The detail title row shows the favorite name, with `Personalize`, `Delete`, and close actions on the same row.
- The detail delete action uses the same inline confirmation panel.
- Clicking `Personalize` enters edit mode for name, summary, ingredients, and steps.
